# gradle获取依赖失败问题

在你没有修改修改过任何代码的情况，你突然发现无法获取某个gradle的依赖，那么一定是***这个依赖在maven仓库中被删除了***。

查看gradle中的maven库配置，确认此maven库中是否可以搜索到***对应版本的依赖***，如果没有，请添加或更换maven库地址。

```groovy
allprojects {
    repositories {
        google()
        jcenter()
        //如果以上maven库无法找到你的依赖，在此配置你的依赖库所在maven地址
    }
}
```
